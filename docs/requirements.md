# Requirements

## 機能要件

### 認証・設定

- **REQ-001**: ユーザーは GitHub Personal Access Token（PAT）を入力・保存できる
- **REQ-002**: PAT は `localStorage` に保存し、ページリロード後も維持される
- **REQ-003**: PAT の有効性を GitHub API（`/user`）で検証できる
- **REQ-004**: PAT を削除・更新できる
- **REQ-005**: 必要スコープ: `repo`（プライベートリポジトリ読み取り）、`read:org`（組織リポジトリ読み取り）

### リポジトリ管理

- **REQ-010**: ユーザーは `owner/repo` 形式でリポジトリを追加できる
- **REQ-011**: 追加したリポジトリは `localStorage` に保存され、ページリロード後も維持される
- **REQ-012**: リポジトリを一覧表示・削除できる
- **REQ-013**: リポジトリが存在しない・アクセス権限がない場合はエラー表示する

### Issue / PR リスト

- **REQ-020**: 設定済みの全リポジトリから Issue と PR を取得してリポジトリごとにグループ表示する
- **REQ-021**: 各グループヘッダーにリポジトリ名（GitHub へのリンク）・件数・折りたたみボタンを表示する
- **REQ-022**: 各アイテムに以下の情報を表示する
  - アイコン（Issue / PR・ステータスの区別、GitHub Octicons SVG）
  - タイトル
  - 番号（`#123`）
  - ステータス（open / closed / merged）
  - ラベル（複数、アプリ固有カラーパレットで色付け）
  - 作成日時
  - 更新日時
  - 関連 Issue（PR 本文の `Closes/Fixes/Resolves #XX` パターンから抽出）
- **REQ-023**: アイテムをクリックすると GitHub の該当ページを新しいタブで開く
- **REQ-024**: グループは折りたたみ・展開できる

### フィルタリング

- **REQ-030**: タイプ（Issue / PR）を複数選択でフィルタリングできる。デフォルトは両方選択。未選択は 0 件表示
- **REQ-031**: ステータス（Open / Closed）を複数選択でフィルタリングできる。デフォルトは Open 選択。未選択は 0 件表示
- **REQ-032**: ラベルでフィルタリングできる（複数選択可）。デフォルトは全選択（空配列 = すべて表示）

### キャッシュ

- **REQ-040**: API レスポンスを `localStorage` にキャッシュする（デフォルト TTL: 5分）
- **REQ-041**: キャッシュが有効な間は API を再リクエストしない
- **REQ-042**: 手動更新ボタンでキャッシュをクリアして再取得できる
- **REQ-043**: キャッシュの最終更新日時を表示する

### 関連 Issue 抽出

- **REQ-050**: PR 本文から以下のパターンを抽出する
  - `Closes #123`, `Fixes #123`, `Resolves #123`（大文字小文字不問）
  - `Closes owner/repo#123` 形式も対応

## 非機能要件

- **NFR-001**: GitHub API レート制限（認証済み: 5,000 req/hour）を超えないよう、キャッシュで不要なリクエストを削減する
- **NFR-002**: 静的エクスポート（`output: 'export'`）でビルド可能であること
- **NFR-003**: GitHub Pages のサブパス（`/github-issue-viewer`）で動作すること
- **NFR-004**: `npm run dev` でルートパス（`/`）として動作すること
- **NFR-005**: TypeScript で型安全に実装すること
