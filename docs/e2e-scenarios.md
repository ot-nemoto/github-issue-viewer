# E2E Scenarios

## 設定機能

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 1 | 設定画面を開く | 初期表示 | PAT 入力欄が空、リポジトリリストが空 |
| 2 | 有効な PAT を入力して検証 | トークン検証 | 「✓ @username として認証済み」と表示される |
| 3 | 無効な PAT を入力して検証 | エラー表示 | 「✗ トークンが無効です」と表示される |
| 4 | `owner/repo` 形式でリポジトリを追加 | リポジトリ追加 | リストに追加され、ページリロード後も維持される |
| 5 | 存在しないリポジトリを追加 | エラー表示 | 「リポジトリが見つかりません」と表示される |
| 6 | リポジトリを削除 | 削除動作 | リストから消え、ページリロード後も反映される |
| 7 | PAT を削除 | PAT 削除 | 入力欄が空になり、ページリロード後もクリアされている |

## メインリスト画面

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 8 | PAT・リポジトリ設定後にメイン画面を開く | データ取得・表示 | リポジトリごとにグループ化されて Issue / PR が表示される |
| 9 | 各アイテムの表示内容を確認 | 表示項目 | グループヘッダーにリポジトリ名・件数、アイテムにアイコン・タイトル・番号・ステータス・ラベル・日付が表示される |
| 10 | アイテムをクリック | リンク動作 | GitHub の該当ページが新しいタブで開く |
| 11 | グループの折りたたみボタンをクリック | 折りたたみ | そのリポジトリのアイテムが非表示になる |
| 12 | 再度折りたたみボタンをクリック | 展開 | アイテムが再表示される |
| 13 | タイプフィルタの「Issue」を外す | フィルタ | PR のみ表示される |
| 14 | タイプフィルタの「PR」を外す | フィルタ | Issue のみ表示される |
| 15 | タイプフィルタを両方外す | フィルタ（未選択） | 0 件表示される |
| 16 | ステータスフィルタの「Closed」を追加選択 | フィルタ | Open + Closed 両方表示される |
| 17 | ステータスフィルタの「Open」を外す | フィルタ | Closed のみ表示される |
| 18 | ステータスフィルタを両方外す | フィルタ（未選択） | 0 件表示される |
| 19 | ラベルフィルタで特定ラベルをクリック | フィルタ | そのラベルを持つアイテムのみ表示される |
| 20 | ラベルフィルタで複数ラベルを選択 | フィルタ（複数） | いずれかのラベルを持つアイテムが表示される |
| 21 | 選択中のラベルを再クリックして全解除 | フィルタ（全解除） | 全アイテム表示される（空配列 = 全選択） |

## キャッシュ・更新

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 22 | 初回表示後にページをリロード | キャッシュ利用 | API リクエストなしで即時表示される（最終更新時刻が変わらない） |
| 23 | 更新ボタンをクリック | 手動更新 | API を再取得し、最終更新時刻が更新される |
| 24 | 5分後にページをリロード（またはキャッシュを手動で古くする） | TTL 期限切れ | API を再取得する |
