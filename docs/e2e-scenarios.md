# E2E Scenarios

## 設定機能

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 1 | 設定画面を開く | 初期表示 | PAT 入力欄が空、リポジトリリストが空 |
| 2 | 有効な PAT を入力して検証 | トークン検証 | 「✓ @username として認証済み」と表示される |
| 3 | 無効な PAT を入力して検証 | エラー表示 | 「✗ トークンが無効です」と表示される |
| 4 | `owner/repo` 形式でリポジトリを追加 | リポジトリ追加 | リストに追加され、ページリロード後も維持される |
| 5 | 存在しないリポジトリを追加 | エラー表示 | 「リポジトリが見つかりません」と表示される |
| 6 | リポジトリを削除 | 削除動作 | リストから消え、ページリロード後も反映される |
| 7 | PAT を削除 | PAT 削除 | 入力欄が空になり、ページリロード後もクリアされている |

## メインリスト画面

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 8 | PAT・リポジトリ設定後にメイン画面を開く | データ取得・表示 | Issue / PR が一覧表示される |
| 9 | 各アイテムの表示内容を確認 | 表示項目 | アイコン・リポジトリ名・タイトル・番号・ステータス・ラベル・日付が正しく表示される |
| 10 | アイテムをクリック | リンク動作 | GitHub の該当ページが新しいタブで開く |
| 11 | タイプフィルタで「Issue」を選択 | フィルタ | Issue のみ表示される |
| 12 | タイプフィルタで「PR」を選択 | フィルタ | PR のみ表示される |
| 13 | ステータスフィルタで「Open」を選択 | フィルタ | open のみ表示される |
| 14 | リポジトリフィルタで特定リポジトリを選択 | フィルタ | 選択したリポジトリのアイテムのみ表示される |
| 15 | ラベルフィルタで特定ラベルを選択 | フィルタ | 選択したラベルを持つアイテムのみ表示される |

## キャッシュ・更新

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 16 | 初回表示後にページをリロード | キャッシュ利用 | API リクエストなしで即時表示される（最終更新時刻が変わらない） |
| 17 | 更新ボタンをクリック | 手動更新 | API を再取得し、最終更新時刻が更新される |
| 18 | 5分後にページをリロード（またはキャッシュを手動で古くする） | TTL 期限切れ | API を再取得する |

## PAT 未設定・リポジトリ未設定

| # | 手順 | 確認観点 | 期待値 |
|---|------|---------|--------|
| 19 | PAT 未設定でメイン画面を開く | 空状態 | 「設定画面でトークンを設定してください」が表示される |
| 20 | リポジトリ未設定でメイン画面を開く | 空状態 | 「設定画面でリポジトリを追加してください」が表示される |
